Реализовать отображение списком (List view) в дополнение к текущему отображению карточками (Board view). Список должен использовать те же данные, что и карточки, но представлять их в табличном формате с колонками: чекбокс, ID задачи, название задачи, исполнитель, дата дедлайна, приоритет, типы и майлстоун. Задачи должны группироваться по статусам колонок (To Do, Doing, Done), при этом каждая группа может быть свернута или развернута.

Визуально список должен соответствовать дизайну из примера - чистый минималистичный интерфейс с четким разделением колонок, аватарами исполнителей, отображением майлстоунов как текста. Навигация между Board и List view должна быть реализована через переключатель в верхней части интерфейса, при этом состояние данных должно сохраняться при переключении между видами.

Технически нужно создать новый компонент для отображения списка, который будет использовать существующие генераторы данных и семантические токены для стилизации. Таблица должна быть адаптивной, поддерживать темную тему и сохранять все интерактивные элементы - чекбоксы, ховеры, тултипы для майлстоунов и других элементов, аналогично карточкам.

Этапы реализации. Первый этап - структура данных и компоненты. Создать компонент ListView, который принимает те же данные что и BoardView. Реализовать функцию группировки задач по колонкам статусов. Создать базовую HTML структуру таблицы с заголовками колонок и строками задач. Интегрировать переключатель между Board и List view в навигацию.

Второй этап - функциональность и интерактивность. Реализовать логику сворачивания и разворачивания групп по статусам. Добавить обработчики для чекбоксов с синхронизацией состояния между видами. Интегрировать существующие компоненты - аватары, лейблы, даты, майлстоуны в строки таблицы. Добавить тултипы для майлстоунов и других элементов аналогично карточкам.

Третий этап - стилизация и визуальная доработка. Создать стили для таблицы с использованием семантических токенов. Реализовать ховеры для строк, адаптивность колонок, правильное выравнивание контента. Добавить поддержку темной темы через существующие токены. Протестировать визуальное соответствие дизайну из примера и согласованность с Board view.
